Cloud Armor Module

Executive Summary

A production-ready Terraform module for deploying Google Cloud Armor security policies to protect HTTP(S) Load Balancers, CDN endpoints, and network load balancers. Provides IP allowlists/denylists, geo-blocking, rate limiting with throttle and ban modes, pre-configured OWASP WAF rule sets with sensitivity tuning, reCAPTCHA bot management, ML-based Adaptive Protection for Layer 7 DDoS defense, and automatic backend service binding. Supports preview/logging-only mode for safe rule rollout.

Component Breakdown

google_compute_security_policy.this

Core security policy with a mandatory default rule (allow or deny) at lowest priority (2147483647). Custom rules use IP matching (SRC_IPS_V1) or CEL expressions for advanced matching. Pre-configured WAF rule sets apply OWASP ModSecurity CRS with tunable sensitivity levels (0-4) and opt-out rule IDs.

Rate Limiting

Throttle and rate-based ban actions with configurable thresholds (count/interval), enforce keys (IP, HTTP_HEADER, HTTP_COOKIE, XFF_IP, HTTP_PATH, SNI, REGION_CODE), and ban durations.

Adaptive Protection

ML-based anomaly detection for Layer 7 DDoS attacks with STANDARD or PREMIUM rule visibility for auto-deployed mitigation rules.

google_compute_backend_service.binding

Automatically attaches the security policy to specified backend services. Uses lifecycle ignore_changes to avoid disrupting existing backend configurations (health checks, protocol, CDN settings).

Data Flow

  • Inbound Request: Traffic hits the HTTP(S) Load Balancer and is evaluated against Cloud Armor rules in priority order.
  • Rule Evaluation: Each rule matches by IP range (SRC_IPS_V1) or CEL expression. First matching rule's action is applied.
  • Rate Limiting: Requests exceeding thresholds are throttled (conform/exceed actions) or banned for configured durations.
  • WAF Inspection: Pre-configured OWASP rules inspect request payloads for SQLi, XSS, LFI, RFI, RCE, protocol attacks.
  • Bot Challenges: Suspicious traffic is redirected to reCAPTCHA for human verification.
  • Adaptive Protection: ML models detect traffic anomalies and auto-deploy mitigation rules.
  • Default Rule: Unmatched traffic falls through to the default rule (allow or deny).

Industry Adaptation

  • Financial Services: IP allowlists for partner access, WAF rules for PCI DSS compliance, rate limiting on API endpoints to prevent abuse.
  • E-Commerce: Bot management with reCAPTCHA during flash sales, geo-blocking for region-specific pricing, adaptive DDoS protection.
  • Healthcare: Strict IP allowlisting for authorized access points, verbose logging for HIPAA audit trails, deny-by-default with explicit allowlists.
  • SaaS Platforms: Per-tenant rate limiting via HTTP headers, OWASP WAF for application protection, preview mode for safe rule testing.

Production Readiness Checklist

  • Deploy WAF rules in preview mode first, review logs before enforcement
  • Enable Adaptive Protection with Layer 7 DDoS defense
  • Configure rate limiting with appropriate thresholds (start permissive, tighten)
  • Set WAF sensitivity level to 1 (low false positives) initially
  • Enable JSON parsing in advanced options for API protection
  • Set verbose logging during initial rollout for visibility
  • Configure ban thresholds for repeat offenders
  • Test reCAPTCHA integration before enabling in production
  • Document all rule priorities and maintain a priority registry
  • Attach policy to all relevant backend services

Deployment

module "cloud_armor" {
  source = "github.com/kogunlowo123/terraform-gcp-cloud-armor"

  project_id          = "my-project"
  name                = "web-security-policy"
  default_rule_action = "deny(403)"

  adaptive_protection_config = {
    enabled                     = true
    layer_7_ddos_defense_enable = true
  }

  rules = [
    {
      action   = "allow"
      priority = 100
      description = "Allow corporate IPs"
      match = {
        versioned_expr = "SRC_IPS_V1"
        config = { src_ip_ranges = ["203.0.113.0/24"] }
      }
    },
    {
      action   = "throttle"
      priority = 2000
      description = "Rate limit API"
      match = {
        expr = { expression = "request.path.matches('/api/.*')" }
      }
      rate_limit_options = {
        rate_limit_threshold = { count = 100, interval_sec = 60 }
        exceed_action = "deny(429)"
        enforce_on_key = "IP"
      }
    }
  ]

  pre_configured_waf_rules = [
    {
      action    = "deny(403)"
      priority  = 3000
      rule_set  = "sqli-v33-stable"
      preview   = true
      sensitivity_level = 2
    }
  ]
}
